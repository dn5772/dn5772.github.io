
# Tap Smith (탭 스미스) — 게임 개발 기획서 (Draft)

## 1. 개요
- **장르**: 무한 강화 시뮬레이션 / 방치형 클리커
- **플랫폼**: Telegram Mini App (HTML5)
- **핵심 컨셉**: 채굴 → 강화 → 판매 → 성장 루프
- **목표**: +50강 이상의 전설 무기 제작

---

## 2. 핵심 루프(Core Loop)
1) **광산 탭**으로 골드 획득  
2) **곡괭이 업그레이드**로 탭당 수익 상승  
3) **대장간에서 강화**로 무기 가치 상승  
4) **판매**로 대량 골드 회수  
5) **최고 강화 기록 갱신** 및 반복

---

## 3. 시스템 설계

### 3.1 무기 시스템
- **자동 지급**: 판매/파괴 시 즉시 0강 무기 지급
- **강화 수치 무한**
- **수식언(프리픽스)**: 강화 구간별 명칭 변경
- **무기 종류**: Sword/Hammer/Club/Bow (티어 배수 적용)

### 3.2 곡괭이 시스템
- **확정 업그레이드(해금)**
- **탭당 골드 획득량 상승**
- **단계 예시**: Wood → Stone → Iron → Gold → Diamond → Emerald → Mana → Nebula

### 3.3 강화 확률
- **+1~+5**: 100% 성공  
- **+6~+15**: 성공 80→40%  
- **+16~+30**: 성공 30→15%  
- **+31~+50**: 성공 10→3%  
- **+51~**: 성공 1% 고정  
- 실패/파괴 확률은 구간별 보간

### 3.4 판매 수익
- **수식**: `기본가 × (강화^3.5) × 티어`
- 고강화 판매가 기하급수적으로 상승

### 3.5 미션/출석
- **일일 출석** 보상  
- **일일 도전 과제**(예: 강화 10회)

### 3.6 친구 초대
- 초대 수에 따라 보상 지급(서버 관리)

---

## 4. UI/UX 구성

### 4.1 탭 구조
- **Market**: 추후 업데이트 안내
- **Mining**: 탭 채굴 + 곡괭이 업그레이드
- **Smithy**: 강화/판매 + 무기 카드
- **Friends**: 초대 버튼/보상 표시
- **Missions**: 출석/일일 도전
- **Settings**: 언어 변경(i18n)

### 4.2 시각 연출
- 강화 수치에 따른 **오라/색/폰트 변화**
- **강화 성공 시 화면 흔들림**
- 강화 버튼 **연타 방지 및 짧은 이펙트**

---

## 5. 경제/밸런스(서버 관리)

### 서버 단일 소스 관리
- 강화 확률/비용/판매가 파라미터
- 곡괭이 단계/비용/성능
- 미션/출석/초대 보상
- 유저 세이브(골드, 무기, 곡괭이, 기록 등)

---

## 6. 기술 설계(현재 구조 기준)

### 6.1 클라이언트
- Telegram Mini App (HTML/JS)
- i18n JSON 분리
- API 연동 전환 예정(로컬 세이브 제거)

### 6.2 서버(.NET)
- 게임 밸런스/확률/세이브 전담
- 강화/판매/업그레이드/미션 로직 처리
- API 중심 구조

### 6.3 TON 연동(향후)
- 지갑 연결(TON Connect)
- 보상 지급/소유권 처리(온체인)
